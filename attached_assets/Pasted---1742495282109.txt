Название проекта:
Система автоматизированного развертывания и управления реверс-прокси

Описание задачи:
Разработать программное обеспечение для автоматизированного развертывания реверс-прокси. Софт должен обеспечивать централизованное управление развертыванием и конфигурацией реверс-прокси на серверах, которые добавляются и настраиваются через административную панель. Приложение должно быть построено на модульной архитектуре, где каждый функциональный блок реализован в виде отдельного модуля.

Технологический стек:

    Язык разработки: Python (рекомендуется использовать последнюю стабильную версию)
    База данных: MySQL
    Операционная система для развертывания: Ubuntu (необходимо создать скрипт для быстрого деплоя)

Требования к архитектуре и реализации:

    Модульная структура:
        Каждый функционал (например, управление серверами, развертывание реверс-прокси, управление доменами, логирование, мониторинг) реализуется в виде отдельного независимого модуля.
        Взаимодействие между модулями должно осуществляться через четко определённые API или интерфейсы, что позволит легко добавлять или изменять функциональность без затрагивания базового кода.

    Управление серверами через админку:
        Реализовать административную панель, позволяющую добавлять, редактировать и удалять сервера, на которых будет развертываться реверс-прокси.
        Для каждого сервера предусмотреть возможность указания необходимых параметров (IP-адрес, SSH-ключ, дополнительные параметры конфигурации и т.д.).
        Разработать механизм проверки доступности сервера и корректности введённых данных.

    Автоматизированное развертывание реверс-прокси:
        После добавления сервера через админку, система должна иметь возможность автоматически установить и настроить реверс-прокси на данном сервере.
        Конфигурация реверс-прокси (маршруты, домены, SSL-терминация и пр.) должна задаваться централизованно через панель управления и применяться к выбранным серверам.
        Реализовать механизм динамической конфигурации, позволяющий вносить изменения «на лету» без необходимости ручного редеплоя.

    Управление доменами и группами доменов:
        Добавить раздел для управления доменами, где можно создавать группы доменов.
        При добавлении каждого домена предусмотреть обязательное указание таргет IP — адреса, на который в итоге должен перенаправляться трафик, приходящий на реверс-прокси.
        Каждая группа доменов должна иметь возможность быть ассоциированной с определённым сервером, на котором будет настроен реверс-прокси для данных доменов.
        Обеспечить возможность легкого добавления, удаления и редактирования доменов и их групп через административную панель.

    База данных:
        Использовать MySQL для хранения конфигурационных данных, логов и статистики.
        Применить ORM (например, SQLAlchemy) для упрощения взаимодействия с базой данных и повышения читаемости кода.

    Скрипт развертывания:
        Разработать bash-скрипт или сценарий с использованием инструментов автоматизации (например, Ansible) для быстрой установки всех зависимостей, настройки MySQL, установки Python-пакетов (через pip и виртуальное окружение) и запуска приложения.
        Скрипт должен корректно работать на чистой Ubuntu-системе и минимизировать ручные действия администратора.

    Документация и тестирование:
        Предоставить подробную документацию для разработчиков и конечных пользователей, включающую описание архитектуры, инструкции по установке, настройке и добавлению новых модулей или серверов.
        Реализовать модульные тесты для каждого функционального блока, чтобы обеспечить корректную работу системы в целом и каждого модуля в отдельности.

Пожелания по реализации:

    Соблюдать принципы чистого и хорошо структурированного кода, снабжённого комментариями, разъясняющими ключевые моменты.
    Использовать виртуальное окружение для Python и файл requirements.txt для управления зависимостями.
    Обеспечить надёжное логирование и мониторинг для оперативного обнаружения и устранения возможных ошибок в работе системы.
